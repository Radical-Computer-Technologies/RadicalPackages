RADCore

RADCore is the foundation module. It provides the event system, connection handles, event loops, timers, cross-thread invocation helpers, IO devices, memory buffers, files, binary data streams, JSON, settings, filesystem utilities, futures, and compact value primitives. Everything lives in the RADCore namespace.

The event system is built on RADObject, the base class that tracks thread affinity and auto-disconnects connections when a receiver is destroyed. Typed events are declared with the RAD_EVENT(name, (Args...)) macro and wired with RADCONNECT/RADCONNECT_QUEUED; Event::connect() delivers directly on the receiver's thread while connect_queued() posts through an explicit EventLoop. Connections are represented by the Connection handle returned from connect().

EventLoop is the task loop, selectable between the locked RADEVENT_LOCK and low-latency RADEVENT_NOLOCK strategies, with post(), postDelayed(), exec(), and quit(). EventLoopHandle is a weak posting handle safe to keep beyond the loop's lifetime, and singleShot(), invoke(), and invokeBlocking() are free-function helpers for cross-thread work. RADElapsedTimer, RADDeadlineTimer, and the RADCancellationSource/RADCancellationToken pair cover timing and cooperative cancellation.

IO is modeled on the abstract RADIODevice, with RADFile and in-memory RADBuffer implementations, OpenMode flags, and RADDataStream for endian-aware binary serialization. Structured returns use RADResult<T> and RADError rather than exceptions. Data primitives include RADVariant, RADDateTime, RADUuid, RADUrl, and the RADJsonValue/RADJsonDocument JSON tree, with RADSettings layering a JSON-file-backed key/value store on top. Filesystem helpers live in the RADCore::FileSystem namespace, alongside RAII RADTemporaryDirectory, RADTemporaryFile, RADLockFile, and the RADFileSystemWatcher (inotify-backed). RADFuture/RADFutureT<T> wrap shared futures for async results.

RADCore also exposes runtime build introspection through runtimeEdition(), runtimeRequiresLicenseKey(), and runtimeAuditStamp() (defined in RADCoreExtras.cpp), which report the edition and license expectations of the loaded library for diagnostics.

Core features

  • RADObject + RAD_EVENT/RADCONNECT typed events with automatic cleanup.
  • EventLoop (locked and lock-free strategies) with EventLoopHandle posting.
  • RADIODevice, RADFile, RADBuffer, and endian-aware RADDataStream.
  • RADJsonValue/RADJsonDocument and JSON-backed RADSettings.
  • RADResult<T>/RADError, plus RADVariant, RADDateTime, RADUuid, RADUrl.
  • RADCore::FileSystem helpers, RAII temp files/locks, RADFileSystemWatcher.

Beta notes:

  • File watching and process/PTY helpers are Linux-first in this beta.
  • RADFuture/RADFutureT and other threaded helpers are excluded from RADLIB_EMBEDDED builds.

Example:

RADCore::EventLoop loop(RADCore::RADEVENT_LOCK);
RADCore::RADFile file("config.json");
if (file.open(RADCore::OpenMode::ReadOnly)) {
auto bytes = file.readAll();
std::string text(bytes.begin(), bytes.end());
if (auto doc = RADCore::RADJsonDocument::fromJson(text)) {
const auto& root = doc->root();
RADInfo() << "port = " << root["port"].toInt(8080);
}
}
RADCore::singleShot(250, loop, [&loop] { loop.quit(); });
loop.exec();
