Packaging

This page covers user-facing package installation and package trust. Maintainer publishing commands are intentionally not part of the public 0.2.1 release guide.

Runtime Packages

  • radbuild: frozen RadBuild CLI and build helpers
  • radbuild-server: optional frozen server and worker commands
  • radbuild-vscode-support: VSCode support files
  • radbuild-radhdl: RadHDL catalog and documentation generator assets
  • radbuild-doc: installed documentation

Install From Downloaded Packages

sudo apt install ./radbuild_0.2.1_amd64.deb
sudo apt install ./radbuild-doc_0.2.1_all.deb

Verify Package Contents

The main runtime package should contain frozen executables and shell launchers, not loose runtime Python source:

dpkg-deb -c radbuild_0.2.1_amd64.deb

Expected shape:

./usr/bin/radbuild
./usr/bin/build_vivado
./usr/lib/radbuild/bin/radbuild
./usr/lib/radbuild/bin/build_vivado

RADix Packages

RADix OS packages use .radpm metadata and archives. RadBuild verifies package index signatures, archive signatures, SHA-256 hashes, and byte counts when resolving package locks.

radbuild packages resolve-radpm --settings settings.terminal.json --json
